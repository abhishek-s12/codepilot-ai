Six services doing the quiet work underneath every question you ask.

src/index/scanner.ts
◆

Recursive scanner

Walks the workspace, skips the noise — node_modules, dist, .git — and extracts size, extension, and line counts as it goes.

src/index/ast-parser.ts
◆

AST-based parsing

Classes, functions, decorators, and imports are discovered structurally, then chunked along their natural boundaries.

src/index/embeddings.ts
●

Local embeddings

Every chunk becomes a 384-dimensional vector on-device, using all-MiniLM-L6-v2 — no embedding API bill, ever.

src/index/vector-store.ts
●

Qdrant vector store

Deterministic collections, one per repository, ranked by cosine similarity when a query comes in.

src/ai/rag.ts
◆

Cited chat

Answers link back to the file and line that produced them, with prompt and completion tokens tracked per turn.

src/ai/actions.ts
✓

One-click actions

Explain, test, refactor, document, audit, and profile a file — seven prompts tuned for a single right-click.

codexa/agent/workspace.tsx

The agent reads the call graph before it writes a line.

Give it a goal — "add email verification" — and it scans imports and call paths to work out exactly which files need to change, then applies patches as a reviewable diff.

  • ✓ Dependency-aware task checklist, generated from your actual imports
  • ✓ Live status per task: todo, running, completed, failed
  • ✓ Every patch lands as a git diff — accept, reject, or edit inline
  • ✓ Interactive call graph to jump straight to a declaration
